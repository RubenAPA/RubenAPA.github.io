Limpieza y Exploración de Datos

Proyecto de SQL "Anonymous_Sales_2019"

Dominio: Transacciones de ventas.
Creado por: Rubén Padilla Aguilar.
Publicado en: 2-5-2023.
Herramientas utilizadas: MS SQL Server y Tableau.
Propósito del Proyecto: Limpieza y exploración de datos con SQL y visualización con Tableau.
Conceptos utilizados: CTE's, manejo de tablas y vistas, modificación de datos, tablas temporales, funciones de agregación, conversión de tipos de datos.

Información del conjunto de datos:
1. El conjunto de datos fue descargado de Kaggle bajo el nombre "SalesAnalysis" por Serdar Ozturk.
2. El conjunto de datos es una compilación de 12 tablas, cada una contiene datos de un mes específico del año 2019.
3. Cada tabla tiene entre 9.000 y 26.000 registros con datos y columnas con las características: Order ID, Product, Quantity Ordered, Price Each, Order Date y Purchase Address.
4. Los datos estaban sucios con valores faltantes y valores no coincidentes.

Problema empresarial:
En un escenario hipotético, una empresa tiene como objetivo analizar una gran cantidad de datos recolectados en el año pasado para poder tomar decisiones basadas en datos relacionados con el comportamiento del cliente. Para completar esta tarea, quieren limpiar los datos y realizar un análisis exploratorio con Microsoft SQL Server para determinar si la información recolectada es útil y relevante. Si es así, quieren crear un hermoso dashboard en Tableau. Para lograr estos diferentes procesos, proporcionan algunos ejemplos de lo que les gustaría saber, como el valor monetario y la cantidad de órdenes procesadas, las 10 mejores ventas y ubicaciones utilizando las variables disponibles y la distribución de ventas por mes, entre otros.

Objetivo del proceso de limpieza:
1. Preparar los datos correctamente para poder implementarlos en los siguientes procesos, como exploración, análisis y visualización.

Objetivos del proceso de exploración:
1. Categorizar información de múltiples variables basadas en su relevancia para el análisis de ventas.
2. Comparar los datos en valor monetario y cuantitativo utilizando diferentes variables como el mes, la ciudad, el producto y así sucesivamente.
3. Distinguir datos importantes como el producto más o menos ordenado, los mejores meses en ventas, entre otros.
4. Cuestionar cómo obtener nuevas ideas, cómo mejorar estos datos en el futuro, etc.
5. Identificar algunas recomendaciones para la empresa hipotética del problema empresarial.

Resultados principales:
1. Se crearon tablas para la cantidad de productos ordenados para cada uno, número de estados desde donde los clientes realizaron pedidos, los 10 productos más ordenados del año, valor y cantidad generados por cada producto según el mes, valor total de cada pedido, dinero generado en pedidos cada mes, valor monetario de cada producto en el estado con el valor monetario general más bajo, y personas con la posibilidad de obtener una regalía.
2. El producto más ordenado del año fue 'AAA Batteries (4-pack)' con una cantidad de 31.017 y las mejores ventas ocurrieron en diciembre.
3. El mes con más dinero generado fue diciembre y el producto más vendido fue la computadora portátil MacBook Pro con una distribución relativa del 23,6%.
4. La computadora portátil MacBook Pro es el producto más caro con una diferencia de precio de $700,01 en comparación con el segundo producto más caro, y las mejores ventas para este producto ocurrieron en diciembre.
5. El estado de California (CA) vendió 11.000 computadoras portátiles MacBook Pro, convirtiéndose en el mejor estado en términos de ventas para este producto.
6. Las personas con la posibilidad de obtener una regalía fueron aquellas que compraron más de una computadora portátil MacBook Pro ya que es el producto más caro. Sus Order_ID son 181069, 181544, 200528 y 210292.
7. El estado con el valor monetario más bajo en términos de pedidos fue Maine (ME).

Recomendaciones finales del análisis:
1. Crear un modelo estrella que recolecte más datos sobre clientes, pedidos, ventas y más.
